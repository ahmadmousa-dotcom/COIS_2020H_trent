Definition

The Binary Tree

Before we can search efficiently we must establish the Binary Tree.

  • Hierarchical data structure.
  • Golden Rule: Every node has at most 2 children.
  • Branches only go downwards — no cycles.
📌 Professor's Note

Think of a family tree inverted — branches go down. "Binary" simply means two — maximum two children per node.

Core Concepts
Structuring for Speed
Rules

Binary Search Trees (Properties)

A Binary Tree becomes a BST when it follows strict ordering rules.

  • Left Subtree: all values strictly less than the root.
  • Right Subtree: all values greater than or equal to the root.
  • Recursive: every subtree must itself be a valid BST.
  • In-order traversal always produces sorted output.

The Problem with Standard BSTs

What happens if we insert data that is already sorted? (e.g., 10, 20, 30, 40 in order)

  • The tree stretches entirely to one side.
  • This is called a Degenerate Tree.
  • Search time degrades from $O(\log n)$ to $O(n)$.

What structure did we accidentally build? A Linked List! Every new node appended to the right — no branching whatsoever.

📌 Professor's Note

1,000 sorted values: degenerate BST needs 1,000 comparisons; a balanced tree needs just ~10.

$$ \text{BF} = \text{Height}(\text{Left}) - \text{Height}(\text{Right}) $$

Golden Rule: Every node's BF must be in {-1, 0, 1}

BST — Find Largest Node

Mirror of Find Smallest — keep going right until you hit a null.

  • The largest node is the rightmost leaf.
  • findLargest(node) recurses on right child.

"Going left finds the minimum, going right finds the maximum."

function findLargestIterative(root):
  if root is null: return null
  current = root
  while current.right is not null:
      current = current.right
  return current

Search

BST Search Algorithm

At each node, compare target to current:

  • $target < current$ → go left
  • $target > current$ → go right
  • $target = current$ → Found!
  • Null reached → Not Found

Node Deletion
Case A

Delete Node — Case A: Leaf

Deletion is the most complex BST operation — we must preserve the ordering property. There are four scenarios.

Case A: The Leaf Node

No children. Set the parent's pointer to null. The garbage collector recycles the memory.

"Case A is like pruning a dead leaf — no structural impact on the rest of the tree."

Ahmad Mousa • COIS 2020H
Algorithms
Node Deletion
Case B

Delete Node — Case B: Only Right Subtree

Case B: Only a Right Child

No left child. Bypass the deleted node by attaching its right subtree directly to its parent. The right subtree is "promoted".

"Case B is like bypassing a broken segment of pipe — the flow continues uninterrupted."

Case C: Only a Left Child

Node Deletion
Case D

Delete Node — Case D: Two Subtrees

Case D: Two Children

Cannot simply bypass. Must find a replacement that preserves BST ordering. Two valid choices:

  • The largest in the left subtree (in-order predecessor)
  • The smallest in the right subtree (in-order successor)

Example: Delete root 23. Largest in left subtree = 20. Copy 20 into root, then delete 20 from left subtree (which is a simpler Case A or B/C).

Putting it Together
Pseudocode

Delete Node Algorithm

The deleteBST algorithm uses recursion to find the target, then applies the correct case.

deleteBST(root, dltKey):
  if root is null: return null

  // Step 1: Navigate to the node
  if dltKey < root.data:
      root.left = deleteBST(root.left, dltKey)
  else if dltKey > root.data:
      root.right = deleteBST(root.right, dltKey)

  // Step 2: Node found! Apply cases
  else:
    // Cases A, B, C: 0 or 1 child
    if root.left is null:
        return root.right   // A or B
    if root.right is null:
        return root.left    // C

    // Case D: Two children
    root.data = findLargest(root.left).data
    root.left = deleteBST(root.left, root.data)

  return root

Case A — Leaf (0 children)

root.left is null AND root.right is null → both returns hit; first one fires: return null

Case B — Only right child

root.left is null → return root.right

Case C — Only left child

root.right is null → return root.left

Case D — Two children

Find largest in left subtree, copy its data, then recursively delete that original node.

The Problem
Vulnerability of BSTs
Performance

Balanced Binary Trees

The primary vulnerability of standard BSTs is that they are not guaranteed to be balanced.

Tree State (1,000 Nodes)Worst-Case Search
Unbalanced (Linked List) $O(n)$ ≈ 1,000 ops
Balanced $O(\log n)$ ≈ 10 ops

Root cause: Insert sorted data → degenerate tree. The BST has no mechanism to fix itself — it degrades silently to $O(n)$.

Ahmad Mousa • COIS 2020H
The Solution
Advanced Structures
Self-Balancing

Making a Tree Self-Balancing

To solve the $O(n)$ worst-case issue, advanced structures establish a core rule that governs their shape during every insert and delete.

AVL Tree

Balance Factor = Height(L) − Height(R) must be in {−1, 0, 1} at every node. Violations trigger rotations immediately.

Guarantee: O(log n) always

2-3-4 Tree

Nodes can hold 1, 2, or 3 keys. All leaves are at the same depth — perfect balance by definition.

Guarantee: O(log n) always

AA Tree

Variant of red-black tree. Only allows right-leaning red links — simpler rotations, easier to implement.

Guarantee: O(log n) always

The Trade-off: Once a balancing rule is established, the algorithm must do extra work (rotations/splits) on every insert and delete to enforce it. This overhead is worth it for systems that perform millions of searches.

Summary

Key Takeaways

🌳

BST Property

Left < root ≤ right at every node. Enables $O(\log n)$ search, insert, and delete on average.

⚠

Degenerate Risk

Sorted insertion creates a linked list. Worst-case degrades to $O(n)$ — unacceptable for large datasets.

⚖

AVL Solution

Balance Factor ∈ {−1, 0, 1}. Four rotation types restore balance. Guarantees $O(\log n)$ always.

🗑

Delete Cases

A: leaf → null. B: only right → promote right. C: only left → promote left. D: two children → replace with in-order predecessor.

AVL Trees — Introduction

The Problem with Plain BSTs

  • BST search is O(h) where h = tree height
  • Random insertions: h ≈ log n → O(log n) average
  • Sorted insertions: h = n → O(n) worst case
  • Inserting 1,2,3,4,5 creates a degenerate linked list
  • AVL trees guarantee height ≤ 1.44 log(n+2) at all times
  • After every insert/delete, tree self-balances via rotations
BALANCED — O(log n)
DEGENERATE — O(n)
📚 Prof Note: Named after Soviet mathematicians Adelson-Velsky and Landis — proposed 1962. First self-balancing BST ever devised.

The Balance Factor

BF(node) = HL − HR
AVL Property: |BF| ≤ 1 for every node   |   H(null) = −1   |   H(leaf) = 0

Left subtree 1 level taller

Even (BF = 0)

Both subtrees equal height

Right-Heavy (BF = −1)

Right subtree 1 level taller

What Changes?
Overview

AVL Operations

✓ UNCHANGED Operations

  • Traversal (inorder, preorder, postorder) — identical to BST
  • Search — same algorithm, just guaranteed O(log n)
  • Min / Max — same as BST
  • Height query — O(1) if stored in node

↻ NEW / MODIFIED Operations

  • Insert + rebalance after insertion
  • Delete + rebalance after deletion
  • rotateLeft(node) — new primitive operation
  • rotateRight(node) — new primitive operation
  • updateBalance(node) — recalculates BF
Key Insight: After every insert/delete, walk BACK UP the tree checking BFs and rotating wherever |BF| > 1. At most O(log n) checks needed.
📚 Efficiency: Rotations are O(1) — they only reassign a constant number of pointers. Insert/delete stays O(log n) total even with rebalancing.

The Code
Implementation

AVL Node Structure

// AVL Node structure in C/C++
struct AVLNode {
    int      data;      // stored value
    AVLNode* left;     // left child ptr
    AVLNode* right;    // right child ptr
    short    balance;   // BF: -1, 0, or +1
};

// Height helper (null-safe)
int height(AVLNode* n) {
    if (n == nullptr) return -1;
    return 1 + max(height(n->left),
                      height(n->right));
}

// Update BF at one node
void updateBalance(AVLNode* n) {
    n->balance = height(n->left)
               - height(n->right);
}

The Algorithm
rotateRight

rotateRight — Step by Step

// rotateRight: LL fix
// Returns new root of subtree
AVLNode* rotateRight(AVLNode* root) {
  // Step 1: Save left child
  AVLNode* tempPtr = root->left;

  // Step 2: Move tempPtr's right to root's left
  root->left = tempPtr->right;

  // Step 3: Make root the right child of tempPtr
  tempPtr->right = root;

  // Step 4: Recalculate balance factors
  // (root first! It is now LOWER in the tree)
  updateBalance(root);
  updateBalance(tempPtr);

  // Step 5: Return new root
  return tempPtr;
}

AVL Trees
Putting It All Together
Full Algorithm

Complete AVL Insert Process

1
Insert as Standard BST — Find correct position using BST comparison rules
2
Set New Node BF = 0 — New leaves are perfectly balanced (no children)
3
Walk BACK UP the Tree — Recurse back toward root, checking each ancestor
4
Recalculate BF at Each Ancestor — BF = H(left subtree) - H(right subtree)
5
|BF| ≤ 1 → Continue Up — Node is still balanced, keep checking ancestors
6
BF = +2 → LL or RL Case — Check left child BF: +1/0 = LL, -1 = RL
7
BF = −2 → RR or LR Case — Check right child BF: -1/0 = RR, +1 = LR
8
After Rotation → Continue Checking — Update parent pointers, continue up to root
// Recursive AVL Insert
AVLNode* AVLInsert(AVLNode* root, int val) {
  if (root == nullptr)  // Base case
    return new AVLNode(val);

  if (val < root->data)
    root->left = AVLInsert(root->left, val);
  else if (val > root->data)
    root->right = AVLInsert(root->right, val);
  else return root; // duplicate

  updateBalance(root); // recalculate BF

  if (root->balance == +2) { // Left heavy
    if (root->left->balance >= 0)  // LL
      return rotateRight(root);
    else {                             // RL
      root->left = rotateLeft(root->left);
      return rotateRight(root);
    }
  }
  if (root->balance == -2) { // Right heavy
    if (root->right->balance <= 0) // RR
      return rotateLeft(root);
    else {                             // LR
      root->right = rotateRight(root->right);
      return rotateLeft(root);
    }
  }
  return root;
}
📚 Recursion Magic: The recursive call naturally unwinds back up to the root. Each call returns the (possibly rotated) subtree root, updating parent pointers automatically.
