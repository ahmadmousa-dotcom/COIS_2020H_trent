Linear Search vs. BST Search

AlgorithmBestAverageWorstn=1,000
Linear SearchO(1)O(n)O(n)~1,000 steps
BST SearchO(1)O(log n)O(n)~10 steps

Key: BST search eliminates half the remaining nodes at each step — like binary search, but on a tree structure.

Why BST is faster
LINEAR
Must check every element — no information about where the target is
BST
Ordering rule means each comparison discards an entire subtree
WORST CASE
BST degrades to O(n) if the tree is unbalanced (like a linked list)

Sorting Algorithms

Definition

A Sorting Algorithm is a step-by-step procedure used to rearrange a large number of items into a specific order—typically numerical or lexicographical (alphabetical).

ASCENDING
1, 2, 3, 4, 5
DESCENDING
5, 4, 3, 2, 1
Why it matters: Efficient sorting is fundamental for optimizing other algorithms (like Binary Search) that require input data to be in a predictable order.

Internal vs External Sort

Is all the data being sorted held in RAM at once, or does the dataset overflow into secondary storage?
RAM — Main Memorylimited capacity (5 slots)
↕
Secondary Storage — Disklarge capacity
Internal Sort
Key Difference
Internal: All data in RAM. Fast random access. Standard algorithms apply directly.
External: Sort small RAM-sized chunks, write sorted chunks to disk, then merge them. Used for databases, large files.

Original — Unsorted

Unstable Sort
Equal keys may reorder on each run
Stable Sort
Equal keys always keep their input order
Key rule: Sort stability refers to the ability of keys to maintain their relative order in the list. A stable sort guarantees it — an unstable sort does not.
