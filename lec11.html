Bubble Sort
Core Concept
Overview

What is Bubble Sort?

  • Easy to implement and understand
  • Generally has pretty poor performance compared to modern algorithms
  • Repeatedly scans the list, swapping adjacent elements that are out of order
  • Continues until a full pass produces no swaps — list is sorted!

The largest (or smallest) values gradually "bubble up" to their correct position at the end of the list with each pass.

Each pass: O(n) comparisons
Up to N passes: O(n) passes
Total: O(n²) worst-case

Bubble Sort — Pseudocode

Standard Implementation
procedure bubbleSort( A : list of sortable items ) n = length(A) repeat // up to N passes swapped = false for i = 1 to n-1 inclusive do if A[i-1] > A[i] then // pair is out of order — swap swap( A[i-1], A[i] ) swapped = true end if end for until not swapped // no swaps = sorted end procedure

Outer loop

Repeats until no swaps occur in a full pass — this is the termination condition.

Inner loop

Scans the array left to right, comparing each adjacent pair A[i-1] and A[i].

Swap flag

swapped tracks whether any swap happened. If false after a full pass, the list is already sorted — early exit.

The n–1 passes come from needing to push the largest element step-by-step all the way to position n−1.

Bubble Sort — Optimisations

Optimisation 1 — Skip the sorted partition

Decrement n by 1 after every pass — the last element is already in place.

Decrement n
procedure bubbleSort( A ) n = length(A) repeat swapped = false for i = 1 to n-1 do if A[i-1] > A[i] then swap(A[i-1], A[i]) swapped = true end if end for n = n – 1 ← skip sorted tail until not swapped end procedure

Optimisation 2 — Track last swap position

After each pass, the new boundary is the index of the last swap — everything after it is already sorted.

Dynamic Boundary
procedure bubbleSort( A ) n = length(A) repeat newn = 0 for i = 1 to n-1 do if A[i-1] > A[i] then swap(A[i-1], A[i]) newn = i ← record last swap end if end for n = newn ← jump directly to boundary until n = 0 end procedure

Insertion Sort
Core Concept
Overview

What is Insertion Sort?

  • Builds the final sorted array one item at a time.
  • Highly efficient for small data sets or arrays that are already partially sorted.
  • Takes an element from the unsorted portion and inserts it into its correct position in the sorted portion.
  • It is a stable, in-place algorithm.

Think of it like sorting playing cards in your hands. You pick up one card at a time and slide it into its correct ordered position among the cards you already hold.

Best Case: O(n) (Already sorted)
Average & Worst Case: O(n²) (Reversed)
Space Complexity: O(1)

Pseudocode

Insertion Sort — Pseudocode

Standard Implementation
procedure insertionSort( A : array of items ) n = length(A) // Assume A[0] is already sorted. for i = 1 to n-1 do key = A[i] j = i - 1 // Shift larger elements to the right while j >= 0 and A[j] > key do A[j+1] = A[j] j = j - 1 end while // Place the key into the empty slot A[j+1] = key end for end procedure

Outer Loop

Iterates from 1 to n-1. The variable key holds the value currently being sorted.

Inner Loop (While)

Scans backwards through the sorted portion (j >= 0). If an element is larger than the key, it copies it one position to the right.

Insertion

Once the inner loop stops (either finding a smaller element or reaching index 0), the key is placed into its correct spot at A[j+1].

Insertion Sort — Traits & Variants

Trait 1 — Highly Adaptive

Insertion sort is "adaptive". If the array is already sorted, the `while` loop condition fails immediately.

O(n) Best-Case Scenario
// If array is [1, 2, 3, 4]
for i = 1 to 3 do
  key = A[i] // e.g., key = 2
  j = 0
  // A[j] is 1. 1 > 2 is FALSE.
  // Loop never runs!
  while j >= 0 and A[j] > key do
    ...
  end while
end for

Variant — Binary Insertion Sort

Instead of scanning backwards linearly, use Binary Search to find the insertion point on the sorted left side.

Fewer Comparisons, Same Shifts
procedure binaryInsertionSort(A)
  for i = 1 to n-1 do
    key = A[i]
    // O(log n) comparisons to find index
    pos = binarySearch(A, 0, i-1, key)

    // Still requires O(n) shifts!
    for j = i-1 down to pos do
      A[j+1] = A[j]
    end for
    A[pos] = key
  end for
end procedure

Selection Sort
Core Concept
Overview

What is Selection Sort?

  • Often considered the simplest of sorts to understand.
  • You subdivide the list into sorted and unsorted parts.
  • You actively search the unsorted part to select the smallest (or largest) element.
  • Once found, you swap it into its final position at the end of the sorted part.

The idea is highly intuitive: "Find the smallest item, put it first. Find the next smallest, put it second."

Best Case: O(n²)
Worst Case: O(n²)
Space Complexity: O(1)

Selection Sort
Mechanism
How it works

Sorted vs Unsorted Partition

  • The list is divided into a sorted (left) and unsorted (right) partition.
  • Pointer i marks the first element of the unsorted partition.
  • We assume the element at i is the minimum (pos_min).
  • Pointer j iterates through the rest of the array. If it finds a smaller element, pos_min is updated.
  • At the end of the pass, we swap the element at pos_min with the element at i.

Position of Min

Tracks the index of the smallest value found so far during the pass.

The Swap

Happens strictly once per pass, adding the absolute minimum to the sorted tail.

Selection Sort — Code Example

Standard C# Implementation
cforbeginners.com
public void selectSort(int[] arr) { // pos_min tracks the index of the minimum value int pos_min, temp; for(int i = 0; i < arr.Length - 1; i++) { pos_min = i; // set min to current index for(int j = i + 1; j < arr.Length; j++) { if(arr[j] < arr[pos_min]) { // Update index when smaller value is found pos_min = j; } } // Only swap if a new minimum was actually found if(pos_min != i) { temp = arr[i]; arr[i] = arr[pos_min]; arr[pos_min] = temp; } } }

Outer Loop

Iterates i up to Length - 1. This marks the growing boundary of our sorted array on the left.

Inner Loop

Scans from i + 1 to the end. It doesn't shift or swap items inside this loop; it purely searches for the lowest value.

The Swap

Occurs only at the very end of the outer loop. We only perform the swap if pos_min changed, confirming we found something smaller.

Selection Sort — Performance

Strictly O(n²) in ALL cases

Selection Sort is entirely completely blind to the existing order of the data. Best case, average case, and worst case are all identical.

Why no "Best Case"?

Even if you pass in a perfectly sorted array like [1, 2, 3, 4, 5], the algorithm doesn't know it's sorted.

To guarantee that 1 is the absolute minimum, the inner loop must still check 2, 3, 4, and 5. This forces it to scan the entire remaining array every single pass.

Redeeming Quality: Minimal Writes

While comparisons are O(n²), the actual number of swaps is at most O(n).

Comparisons vs. Swaps
Algorithm Swaps (Worst)
Bubble Sort O(n²)
Insertion Sort O(n²) (shifts)
Selection Sort O(n)
Takeaway: If you are working on hardware where writing to memory is significantly more expensive than reading from it (e.g., certain types of EEPROM or flash memory), Selection Sort has a distinct advantage over Bubble and Insertion sort.

Core Concept
Overview

What is Merge Sort?

  • A classic Divide and Conquer algorithm.
  • Divides the array into n sublists, each containing 1 element (a list of 1 element is inherently sorted).
  • Repeatedly merges sublists to produce new sorted sublists until there is only 1 sorted list remaining.
  • A highly popular and reliable choice for sorting large datasets.

Instead of struggling to sort a massive list all at once, Merge Sort breaks the problem down into trivial pieces, then intelligently stitches them back together.

Best & Worst Case: O(n log n)
Space Complexity: O(n) (Requires extra memory)
Stability: Stable

The Two Phases: Divide & Merge

1. The Divide Phase

Find the midpoint of the array and split it into two halves. Recursively repeat this process on each half until every sublist contains exactly 1 element.

2. The Merge Phase

Take two adjacent sorted sublists and merge them into a single sorted list. Repeat this bottom-up until the entire array is reconstructed.

Merge Sort — Code Architecture

Pseudocode Structure
procedure mergeSort(arr, left, right) if left >= right return // Base case: 1 element is sorted mid = (left + right) / 2 // Recursively sort first half mergeSort(arr, left, mid) // Recursively sort second half mergeSort(arr, mid+1, right) // Merge the two sorted halves merge(arr, left, mid, right) end procedure

The Base Case

left >= right means the sublist has shrunk to 1 element. The recursion stops and unwinds.

The Recursive Calls

The algorithm calls itself on the left half, then the right half, driving deep down the tree until everything is split up.

The Heavy Lifting

The actual sorting logic lives entirely inside the merge() helper function, which handles the O(n) array stitching at each level.

Merge Sort — Key Traits

Rock Solid O(n log n) Performance

Because the list is always divided strictly in half, the recursion tree is perfectly balanced. It takes log n levels to split, and merging at each level takes O(n).

Not In-Place! (The Tradeoff)

Merge Sort's biggest flaw is its Space Complexity.

To safely merge two halves without overwriting data, it requires a temporary array. This means sorting an array of 1,000,000 items requires allocating memory for another 1,000,000 items. Memory footprint: O(n).

Stability

Merge sort is naturally stable. When comparing elements during the merge phase (e.g., arr[i] <= arr[j]), we pull from the left half first if they are equal, preserving their original relative order.

When to use Merge Sort?
Scenario Suitability
Massive Datasets Excellent
Linked Lists Perfect (No random access needed)
Strict Memory Limits Poor (Use Quick or Heap sort)

Quick Sort
Core Concept
Overview

What is Quick Sort?

  • Like Merge Sort, it is a Divide and Conquer algorithm.
  • It recursively divides the array into sub-parts and sorts those sub-arrays.
  • You select one element to act as a Pivot.
  • The list is divided into three parts:
    • Elements less than the pivot (moved to the left)
    • The Pivot itself (now in its final sorted position)
    • Elements greater than the pivot (moved to the right)
  • Then, recursively quicksort the left part, followed by the right part.

Best & Average Case: $O(n \log n)$
Worst Case: $O(n^2)$ (if a bad pivot is chosen)
Space Complexity: $O(\log n)$ (In-place sorting!)

PIVOT SELECTION STRATEGIES

How do we choose the pivot? The choice heavily impacts performance.

1. Last Element / First Element

Easiest to code (used in our example), but triggers worst-case $O(n^2)$ if the array is already sorted.

2. Random Element

Pick a random index. Avoids consistent worst-case scenarios.

3. Median-of-Three

Look at the far left, far right, and middle elements, and use their median value. Highly optimal!

Lecture 8 — Searching & Sorting
Quick Sort
Mechanism
How it works

The Partition Phase

The magic of Quick Sort happens inside the Partition step. Assuming we pick the Last Element as the pivot:

  • We track the boundary of elements smaller than the pivot using pointer i.
  • We traverse the array with pointer j.
  • If arr[j] < pivot, we expand our "smaller" boundary (i++) and swap arr[i] with arr[j].
  • Once j reaches the end, we swap the pivot into position i + 1.
  • The pivot is now exactly where it belongs in the final sorted array!

Quick Sort — Code Architecture

1. The Recursive Driver
/* low -> Starting index, high -> Ending index */ void quickSort(int[] arr, int low, int high) { if (low < high) { // pivot is the partitioning index // arr[pivot] is now at right place int pivot = Partition(arr, low, high); quickSort(arr, low, pivot - 1); // Before pivot quickSort(arr, pivot + 1, high); // After pivot } }

Base Case Check

if (low < high) ensures we only try to sort lists of 2 or more elements. If not, there are no more elements to sort in that branch.

2. The Partition Logic
static int Partition(int[] arr, int low, int high) { // Choose the pivot (last element here) int pivot = arr[high]; // Index of smaller element. int i = low - 1; // Traverse arr and move smaller elements to left for (int j = low; j <= high - 1; j++) { if (arr[j] < pivot) { i++; Swap(arr, i, j); } } // Move pivot to its correct sorted position Swap(arr, i + 1, high); return i + 1; // return the Pivot position }

Quick Sort — Key Traits

In-Place Sorting (Massive Advantage!)

Unlike Merge Sort which requires $O(n)$ extra memory to merge arrays, Quick Sort swaps elements within the original array, keeping its memory footprint to $O(\log n)$ for the recursion stack.

The "Worst Case" Flaw

If you always pick the Last Element as the pivot, and the array is already sorted, your pivot is always the largest item.

This causes the array to divide into sizes of N-1 and 0 instead of splitting in half. The recursion tree becomes a straight line, degrading performance to a terrible $O(n^2)$. Randomizing the pivot prevents this!

Unstable

Quick sort is naturally unstable. Elements of equal value can have their relative order swapped around wildly as items are thrown back and forth across the pivot boundary.

Searching Algorithms

A Searching Algorithm is a step-by-step procedure designed to locate a specific target element within a larger collection of data.

It is one of the most fundamental operations in computer science. The efficiency of a search depends heavily on the chosen data structure and whether the underlying data is organized (sorted) or unorganized (unsorted).

Lecture 8 — Searching & Sorting
Linear Search
Concept Breakdown
Overview

What is Linear Search?

  • The most straightforward and intuitive searching method.
  • Scans the array sequentially from the very beginning to the end.
  • Checks each element one by one until the target value is found or the list ends.
  • Major Advantage: The array does not need to be sorted. It works on any raw data.

Think of it like looking for a specific book on an unorganized shelf by checking the spine of every single book from left to right.

Best Case: $O(1)$ (Target is the first item)
Worst Case: $O(n)$ (Target is the last item, or not present)
Space Complexity: $O(1)$

The Trade-off

Because it checks every element, it is incredibly inefficient for large datasets. Searching a database of 1,000,000 users might take 1,000,000 checks in the worst case.

Linear Search — Code Example

Standard Implementation
public int LinearSearch(int[] arr, int target) { // Traverse the array from start to finish for (int i = 0; i < arr.Length; i++) { // If the current element matches the target if (arr[i] == target) { return i; // Return the index where it was found } } // If the loop completes without returning, // the target is not in the array. return -1; }

The Loop

A simple for loop controls the scan. It guarantees that we don't skip any elements and stops automatically at the end of the array.

Early Exit

The moment the if statement evaluates to true, the method returns. It does not waste time checking the rest of the array.

Returning -1

It is a standard programming convention to return -1 when a search fails, because -1 can never be a valid array index.

Concept Breakdown
Overview

What is Binary Search?

  • A highly efficient search algorithm that follows the Divide and Conquer paradigm.
  • Strict Requirement: The array MUST be sorted beforehand.
  • It compares the target value to the middle element of the array.
  • If they aren't equal, the half in which the target cannot lie is eliminated, and the search continues on the remaining half.

Think of searching for a word in a physical dictionary. You don't read page 1, then page 2. You open to the middle. If your word comes alphabetically later, you ignore the entire first half of the book.

Best Case: $O(1)$ (Target is exactly in the middle)
Worst Case: $O(\log n)$ (Halves the dataset each step)
Space Complexity: $O(1)$ (Iterative approach)

The Power of Log(n)

Because it halves the search space every single step, searching an array of 1,000,000 items takes at most 20 comparisons. Linear search could take 1,000,000!

Binary Search — Code Example

Iterative Implementation
public int BinarySearch(int[] arr, int target) { int low = 0; int high = arr.Length - 1; // Loop until the boundaries cross while (low <= high) { // Calculate mid to prevent integer overflow int mid = low + (high - low) / 2; if (arr[mid] == target) return mid; // Target found! if (arr[mid] < target) low = mid + 1; // Search right half else high = mid - 1; // Search left half } return -1; // Target not found }

The Boundaries

low and high act as walls containing the active search space. The loop continues as long as these walls haven't crossed over each other.

Calculating Mid

low + (high - low) / 2 is mathematically identical to (low + high) / 2, but is used in professional code to prevent integer overflow errors on massive arrays.

Elimination

Adjusting low = mid + 1 or high = mid - 1 instantly discards half of the remaining array without needing to actively delete or modify any data.
