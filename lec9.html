What Is a Tree?

📐 Formal Definition

A collection of nodes connected by edges that simulates a hierarchical tree structure, with a root value and subtrees of children.

🔵
Non-Linear Unlike arrays
🏛️
Hierarchical Parent → Child
🔍
Searchable Fast navigation

💡 Constraint Example: In a Binary Search Tree, everything to the left < node, and everything to the right > node.

Tree Terminology

🌱
Root
The topmost node of the tree. Has no parent. Exactly one root per tree.
🍃
Leaf
A node with no children (outdegree = 0). Also called an External Node.
👨‍👧
Parent
A node with one or more children directly connected below it.
👶
Child
A node connected directly below a parent. It has exactly one parent.
🔷
Internal Node
A node that is neither a leaf nor the root. Has both a parent and child.
👥
Siblings
Two or more nodes that share the exact same parent node.
⚡

Key Rule: A valid tree has exactly one root and only one unique path from the root to any node.

Tree Terminology — Measurements

🛤️
Path
A sequence of nodes where each is adjacent. Only one path exists from root to node.
📏
Depth / Level
Distance from root = number of edges to reach node. The Root is depth 0.
📐
Height
Level of the deepest leaf in the longest path from root. Empty tree height = -1.
🎯
Degree
Number of direct children a node has. Tree overall degree = max node degree.

Tree Varieties

Examples of Tree Structures

🌲 Binary Tree

Each node has at most 2 children (Left & Right).

💡 Max degree = 2. The foundation for Binary Search Trees and Heaps.

⚖️ Balanced Tree

Left & right subtree heights differ by ≤1.

✅ Both sides have similar depth. Guarantees fast O(log n) search times.

⚠️ Unbalanced

One subtree is significantly deeper than the other.

❌ Left subtree is too deep. Performance degrades to a slow O(n) linked list.

How to Visit Nodes

Tree Traversals

A traversal visits each node exactly once. The order in which we visit them defines the traversal type.

🔽 Depth-First (DFS)

Go as deep as possible down a branch before backtracking. Three variants:

Pre-order (N-L-R)
Node → Left subtree → Right subtree
In-order (L-N-R)
Left subtree → Node → Right subtree
Post-order (L-R-N)
Left subtree → Right subtree → Node

➡️ Breadth-First (BFS)

Visit all nodes level by level — top to bottom, left to right. Uses a queue.

BFS: Level by Level

Breadth-First Traversal (BFS)

How it Works

Uses a queue to process nodes level by level:

  1. Enqueue the root
  2. Dequeue a node, visit it
  3. Enqueue its children (left then right)
  4. Repeat until queue is empty

💡 Use case: Finding shortest path, level-order printing, checking if a tree is complete.

Common Tree Operations

🔍
Search
Find a specific value (BST).
def search(node, val):
  if node is None or node.val == val:
    return node
  if node.val < val:
    return search(node.right, val)
  return search(node.left, val)
📋
Traverse
Visit all nodes (In-order).
def in_order(node):
  if node:
    in_order(node.left)
    print(node.val)
    in_order(node.right)
➕
Insert
Add maintaining order.
def insert(node, val):
  if not node: return Node(val)
  if val < node.val:
    node.left = insert(node.left, val)
  else:
    node.right = insert(node.right, val)
  return node
➖
Delete
Remove node and reconnect.
# 1. Search for node
# 2. If leaf -> remove
# 3. If 1 child -> bypass
# 4. If 2 children -> replace
#    with In-Order successor,
#    then delete successor.
✂️
Prune
Remove an entire subtree.
def prune_left(node):
  if node:
    # GC will auto-delete
    node.left = None
  return node
🌿
Graft
Attach a subtree.
def graft(target, subtree):
  if target.right is None:
    target.right = subtree

Searching a BST

🔍 Search for Value '12'

Because the tree is ordered (Left < Root < Right), we can eliminate half the tree at every step. This is identical to a Binary Search algorithm.

def search(node, val):
  # Base Case
  if node is None: return False
  if node.val == val: return True

  # Recursive Step
  if val < node.val:
    return search(node.left, val)
  else:
    return search(node.right, val)

Pruning & Grafting

✂️ Prune & 🌿 Graft

Unlike arrays where shifting data is slow, moving massive amounts of data in a tree is instantaneous. You just change a single pointer!

Pruning (Cut)

Setting A.left = Null instantly disconnects the entire B subtree.

Grafting (Paste)

Setting X.right = B instantly attaches the massive subtree to node X.

What is a Heap?

⚙️ Two Required Properties

Rule 1 — Nearly Complete Tree
All levels fully filled except the last, which is filled left to right.
Rule 2 — Heap Order (max-heap)
For every node i: a[Parent(i)] ≥ a[i]
The root holds the maximum value.

Max-Heap vs Min-Heap

Heap Types & Applications

🔼 Max-Heap

Root = maximum. Every parent ≥ its children.

🔽 Min-Heap

Root = minimum. Every parent ≤ its children.

📋 Priority Queue

Always access the highest (or lowest) priority item in O(1). Enqueue/dequeue in O(log n).

📊 HeapSort

O(n log n) guaranteed for all cases. In-place, O(1) extra space.

🔍 K-th Largest

Build a min-heap of size K; root = K-th largest element efficiently.

Reheap Up — After Insertion

How it Works

  1. Place new node at the last leaf (end of array)
  2. Compare with its parent
  3. If new node > parent → swap
  4. Repeat until node ≤ parent or at root
Step 0 — 25 just inserted at last leaf (child of 12)
0 / 3

Reheap Down — After Deletion

How it Works

  1. Replace root with the last element
  2. Remove last element from heap
  3. Compare with largest child
  4. If node < largest child → swap
  5. Repeat until heap property restored

Build Heap (Heapify)

Convert an unsorted array into a heap: insert one element at a time + reheap up.

📋 InsertHeap Algorithm

InsertHeap(heap, last, data):
  if heap full → return false
  last = last + 1
  heap[last] = data
  reheapUp(heap, last)
  return true

⏱️ Complexity

Repeated insertion: O(n log n)
Optimised bottom-up: O(n)

🔄 Example — [8, 19, 23, 32, 45, 56, 78]

Start: [8]
Insert 19: 19>8 → swap → [19, 8]
Insert 23: 23>8 → [23, 8, 19]
Insert 32: bubbles up → [32, 23, 19, 8]
Insert 45: → [45, 32, 19, 8, 23]
Insert 56: → [56, 32, 45, 8, 23, 19]
Insert 78: → [78, 32, 56, 8, 23, 19, 45] ✔

HeapSort

Heap Sort

🔢 Two-Phase Algorithm

Phase 1 — Build Heap
Insert each element + ReheapUp → max-heap
Phase 2 — Extract & Sort
Swap root ↔ last → shrink heap → ReheapDown → repeat

⏱️ Complexity

Time: O(n log n) — worst, avg & best
Space: O(1) — in-place

heapSort(heap, last):
  walker = 1

  // Phase 1: build max-heap
  while walker <= last:
    reheapUp(heap, walker)
    walker = walker + 1

  sorted = last

  // Phase 2: sort in place
  while sorted > 0:
    exchange(heap[0], heap[sorted])
    sorted = sorted - 1
    reheapDown(heap, 0, sorted)

  return
