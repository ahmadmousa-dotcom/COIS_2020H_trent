It is important to understand the distinction between a logical data structure and its underlying implementation.

  • Implementation (Hardware): The physical storage mechanism. In this case, a fixed-size Array.
  • Logic (Software): The rules we follow when deciding where to put data.

What is a List?

A list is a collection of objects. It is the starting point for all data structures in this course.

Ordering

Can be Ordered (sorted) or Unordered (random insertion).

Sizing

Can be Fixed Size (classic array) or Dynamic Size (growable).

Complexity

Can contain complex objects, metadata, or even be a List of Lists.

Key List Operations

Every robust list implementation usually supports the following:

  • Create: Initialize
  • Grow: Increase capacity
  • AddOne: Append to end
  • InsertOne: Place at index
  • DeleteOne: Remove item
  • DeleteAll: Clear list
  • Search: Find item
  • Traverse: Print/Visit all
  • Sort: Order items
  • Randomize: Shuffle
  • Merge: Combine lists
Operations dictate performance requirements

Generic Array Implementation

Using Generics <T> allows the list to handle any data type.

public class GenericArray<T> {
    private T[] array; // The actual array

    // Constructor
    public GenericArray(int size) {
        array = new T[size];
    }

    public void setItem(int index, T value) {
        array[index] = value;
    }
    
    public T getItem(int index) {
        return array[index];
    }
}

Why Generics?

Allows code reuse. You can create a GenericArray<int> or GenericArray<string> using the same class definition.

Access is \(O(1)\) if index is known.

The "Grow" Operation

Arrays have a fixed size. If you try to add an element beyond the size, the program crashes:.

To "grow" an array:

  1. Create a new, larger array.
  2. Copy all elements from the old array to the new one.

Complexity: Access is \(O(1)\), but Growing is \(O(N)\).

Resize Strategy

A common strategy is to double the size when full.

public void Grow(int newSize) {
   // Resize is O(N) cost
   Array.Resize(ref array, newSize); 
}

Warning: Doubling very large arrays can cause memory issues.

Grow is a main limitation of arrays

Insertion & Shifting

Add vs. Insert: Adding typically happens at the end. Inserting happens between existing values[cite: 144, 145].

The Shift Problem

If you insert at index [2], you cannot just overwrite. You must shift existing values at [2], [3], ... [n] to the right to make room[cite: 160].

Shift Right \(\rightarrow\)
[A][B][C][D][_]
Insert 'X' at [2]
[A][B][X][C][D]
Insertion in arrays is computationally expensive

Deleting Elements

Deleting from an array introduces holes or requires shifting.

Simple Delete

Set array[i] = null.
Issue: Leaves a "hole" in the list[cite: 201, 202].

Delete from End

Set array[n] = null.
Efficient: No shifting required[cite: 223].

Delete from Front

Requires shifting all subsequent elements left (or using a Circular Array)[cite: 226, 227].

Note: Deleting the structure (the list) is different from deleting the objects (data) it contains. Objects may remain in memory if referenced elsewhere[cite: 347, 350].

Shift Left \(\leftarrow\) is required to close holes

Search & Sort

Search

Linear Search: Check every element one by one.

for(int i=0; i<len; i++) {
  if(array[i].Equals(val)) 
     return true;
}

Simple but slow for large datasets[cite: 236].

Bubble Sort

Repeatedly swap adjacent elements if they are in the wrong order.

  • Easy to implement[cite: 293].
  • Slow: \(O(n^2)\) complexity[cite: 301].
  • Not suitable for large arrays[cite: 295].
More efficient algorithms (Binary Search, QuickSort) covered later.
