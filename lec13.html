Balance Factors [5 marks]
BF = h(left) − h(right)

Computing Balance Factors

The Method: Count the height of each subtree (longest path to a leaf). BF = height(left subtree) − height(right subtree). A null subtree has height 0.

Nodeh(left)h(right)BF Answer
48 (root)330

Step-by-step for node 48:

• Left subtree (22): height = 1 + max(h(12), h(34)) = 1 + max(1, 2) = 3

• Right subtree (72): height = 1 + max(h(58), h(86)) = 1 + max(2,1) = 3

• BF = 3 − 3 = 0 ✓ (balanced)

AVL rule: A tree is AVL-balanced if every node has |BF| ≤ 1. All nodes here satisfy this → the tree is AVL.

BST property: left < node < right

Insertion Paths & AVL Check

Method: Start at root. At each node: if value < node → go left; if value > node → go right. Stop when you reach a null child that's the insertion point.

Part A(c) After inserting 15, is the tree still AVL?

✅ YES the tree remains AVL-balanced.

Reasoning: 15 is inserted as right child of 12. Check the ancestors:

• Node 12: BF changes 0→−1 (right child added). |−1| ≤ 1 ✓

• Node 22: h(left)=2, h(right)=2 → BF = 0. Was −1, now 0 ✓

• Node 48: h(left)=3, h(right)=3 → BF = 0 ✓

No node has |BF| > 1 → tree is still AVL.

Key insight: Inserting 15 actually improved the balance at node 22 (from −1 to 0), because the left subtree grew to match the right subtree height.

Question 1 Part C
AVL Rotations Theory [12 marks]
LL · RR · LR · RL

AVL Imbalance Types & Rotations

How to identify the imbalance type:
1. Find the first ancestor with |BF| > 1 (the critical node z).
2. Look at its tallest child (y) and tallest grandchild (x).
3. The path z→y→x gives the type.

TypePath (z→y→x)Fix
LLleft → leftSingle Right rotation at z
RRright → rightSingle Left rotation at z
LRleft → rightLeft rotate y, then Right rotate z
RLright → leftRight rotate y, then Left rotate z

Memory trick: The imbalance type names the heavy path. The fix is the opposite rotations. Double cases (LR/RL) always need 2 rotations.

Question 2 Part A
Big-O Complexity [8 marks]
2 marks each

Determine the Tightest Big-O

Fragment 1 O(n²)

for i ← 1 to n
  for j ← 1 to i
    print(i, j)

Inner loop runs 1+2+3+…+n = n(n+1)/2 total times → O(n²)

Fragment 3 O(n³)

for i ← 1 to n
  for j ← 1 to n
    for k ← 1 to n
      A[i][j] ← A[i][j] + B[i][k] * C[k][j]

Three independent nested loops, each 1→n → n × n × n = O(n³) (matrix multiply)

Fragment 2 O(log n)

i ← n
while i > 1
  i ← floor(i / 3)
  count ← count + 1

Divides by 3 each step. Reaches 1 after log₃(n) steps → O(log n)

Fragment 4 O(n log n)

gap ← n
while gap > 1
  gap ← floor(gap / 2)    ← log n iterations
  for i ← 1 to n − gap  ← O(n) per iteration
    compare(A[i], A[i+gap])

Outer while: log n times. Inner for: O(n) each time → O(n log n)

Key rules:   Halving/tripling → log n  |  k nested equal loops → O(nᵏ)  |  log-outer × linear-inner → O(n log n)  |  Triangular sum → O(n²)

Sorting Comparison Table [10 marks]
Must memorize!

Sorting Algorithms Complete Reference

Algorithm Best Case Average Case Worst Case Stable? In-place?
Bubble Sort O(n) O(n²) O(n²) ✅ Yes ✅ Yes
Insertion Sort O(n) O(n²) O(n²) ✅ Yes ✅ Yes
Selection Sort O(n²) O(n²) O(n²) ❌ No ✅ Yes
Merge Sort O(n log n) O(n log n) O(n log n) ✅ Yes ❌ No

Best = O(n) means: the algorithm detects an already-sorted array early. Bubble & Insertion can exit early with a flag; Selection always scans everything.

Stable = equal elements preserve their original order. Selection sort is NOT stable because it swaps non-adjacent elements.

In-place = uses O(1) extra space. Merge sort needs O(n) auxiliary space for the merge step → NOT in-place.

Question 2 Part C
Insertion Sort Pseudocode [7 marks]
Write from memory on exam

Insertion Sort Full Solution

The idea: Like sorting playing cards. Take each card (key), slide it left past larger cards until you find its correct spot.

// Insertion Sort   A[0..n-1]
for i ← 1 to n−1:          // outer loop
  key ← A[i]               // element to insert
  j ← i − 1               // start comparing left
  
  while j ≥ 0 and A[j] > key:
    A[j+1] ← A[j]          // shift right
    j ← j − 1
  
  A[j+1] ← key             // place key

Trace: [5, 3, 1, 4]

i=1: key=3, shift 5 → [3,5,1,4]

i=2: key=1, shift 5,3 → [1,3,5,4]

i=3: key=4, shift 5 → [1,3,4,5] ✓

Time Complexity Analysis

CaseComplexityWhy?
Best Case O(n) Array already sorted the while condition fails immediately every time. Only the outer loop runs → n−1 comparisons.
Average Case O(n²) On average, each element shifts half the sorted portion → ~n²/4 total shifts.
Worst Case O(n²) Reverse-sorted array element i must shift past all i elements before it. Total = 1+2+…+(n−1) = n(n−1)/2 = O(n²).

Exam checklist for pseudocode marks: ✓ Outer loop starts at 1   ✓ key = A[i]   ✓ j starts at i−1   ✓ while condition has j≥0 AND comparison   ✓ shift: A[j+1]=A[j]   ✓ final A[j+1]=key

OOP & Fundamentals

MC Answers Q1 to Q13

1. A class represents: b) blueprint for objects
A class is the template; objects are the instances.
2. Purpose of a constructor: b) initialize objects
Constructors run on object creation to set initial state.
3. Keyword that groups related classes in C#: c) namespace
Namespaces organize classes, avoid name conflicts.
4. Underlying structure of an array-based list: b) array
Array-based list stores elements in a contiguous block.
5. Encapsulation is best described as: b) bundling data and methods, controlling access
Private fields + public methods = encapsulation.
6. A queue operates under which principle: b) FIFO
First-In, First-Out. Think a lineup at a store.
7. What is an object in OOP: b) an instance of a class
Dog d = new Dog() → d is the object/instance.
8. Abstraction means: b) hiding unnecessary details, exposing essentials
You use a car without knowing the engine internals.
9. Access modifier restricting to same class: b) private
private < protected < internal < public (visibility scope).
10. Removing from the front of a queue is: c) dequeue
enqueue = add to back; dequeue = remove from front.
11. Logical difference between list and sorted list: b) insertion logic
Sorted list finds the correct position before inserting.
12. LIFO refers to: b) last in first out
Stack structure. Like a stack of plates last placed = first removed.
13. Generics in C# provide: b) type safety with flexibility
List<T> works with any type while staying type-safe at compile time.

Question 3 Multiple Choice
Questions 14–25  |  2 marks each
Data Structures

MC Answers Q14 to Q25

14. Purpose of a hash function: b) map a key to an index or bucket
hash(key) → index. Enables O(1) average lookup.
15. A singly linked list node contains: c) data and next reference
Each node has value + pointer to next node (or null).
16. Inserting at the front of a linked list requires: b) updating the head reference
New node → newNode.next = head → head = newNode. O(1).
17. Polymorphism allows: a) one interface to represent different underlying forms
Animal a = new Dog() same interface, different behaviour.
18. A sentinel node is used to: b) simplify edge cases
Dummy head/tail removes special-case code for empty lists.
19. Operation that removes from a stack: b) pop
push = add; pop = remove; peek = view top without removing.
20. A collision in a hash table occurs when: a) two keys map to the same location
Resolved by chaining or open addressing.
21. A circular linked list differs because: b) the tail points to the head
Last node's next = first node, forming a cycle.
22. Linked-list stack (push at head), pop occurs at: c) head
Push and pop both happen at the head for O(1) performance.
23. Array-based stacks often push at the end because: b) to avoid shifting elements
Appending to end = O(1). Inserting at front requires O(n) shifts.
24. A doubly linked list node contains: c) data, next, and previous
Two pointers allow O(1) traversal in both directions.
25. Deleting a node in a doubly linked list requires: a) updating neighboring pointers
prev.next = node.next; node.next.prev = prev.
